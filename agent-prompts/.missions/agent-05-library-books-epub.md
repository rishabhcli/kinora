- **EPUB ingest ALREADY EXISTS** — `backend/app/ingest/epub_extract.py` (PyMuPDF EPUB→PDF normalization; `looks_like_epub`, `epub_to_pdf_bytes`, `extract_epub_metadata`, `extract_epub_cover`). The upload endpoint `POST /api/books` already accepts `application/epub+zip`. You are **leveraging and hardening** this, not building EPUB parsing from scratch.
- **Existing seeders:** `backend/scripts/seed_public_domain_direct.py` (downloads Gutenberg EPUBs → direct repo writes → **zero model spend**, idempotent, fixed ids `pubdom{gid}`) and `seed_public_domain.py` (HTTP flow, slow, hits image quota). `make seed-demo` seeds the demo book. Study these — your 100-book seeder should follow the **direct, zero-spend** pattern.
- **Covers today are weak:** there is **no `cover_url` field** on the `Book` model (`backend/app/db/models/book.py`). A cover is whatever EPUB declares, stored at `covers/{book_id}` and surfaced only as 'page 1 image' via `GET /api/books/{id}/pages/1`. You will add a **real cover field + endpoint + HD cover sourcing**.
- Upload caps: 50 MB/file, 300 pages/book (to bound DashScope spend). Keep these.
- `KINORA_LIVE_VIDEO` is **OFF** and stays off — seeding must not trigger video spend. Ingest still calls DashScope **image** gen (keyframes/identity) which can `429` — your **direct** seeder must bypass that (write rows directly, like `seed_public_domain_direct.py`), so 100 books cost ~zero.
- Backend tests: isolated DB (`kinora_conflict_test`) + redis db 15, never live `kinora`/db 0. Postgres host port **5433**. Read `CLAUDE.md`.
- Sourcing: **Project Gutenberg** (EPUBs at `https://www.gutenberg.org/ebooks/{id}` / `gutenberg.org/cache/epub/{id}/pg{id}.epub`; bulk metadata via the gutendex API `https://gutendex.com`) and **Standard Ebooks** (beautifully typeset EPUBs) are your primary public-domain sources. HD covers: **Open Library Covers API** (`covers.openlibrary.org/b/id/{id}-L.jpg`), Google Books, or Wikimedia — copyrighted covers are acceptable per the product owner; prefer the highest resolution available. ---

## THE BUILD — WORKSTREAMS

### WS1 — Source & ingest 100+ novels (zero-spend, idempotent) Build `seed_library_100.py`: pull a curated, deduplicated set of **100+ canonical public-domain novels** (mix of Gutenberg + Standard Ebooks; span genres/eras — classics, gothic, sci-fi, adventure, romance, children's, poetry, world literature). Download EPUBs into `assets/books/public-domain/`, write a `catalog.json` manifest, and seed via the **direct repo path** (zero model spend, fixed/stable ids, fully idempotent re-runs). Paginate via the existing EPUB→PDF normalization; respect the 300-page cap (truncate or mark long books). Acceptance: a fresh run produces ≥100 `READY`-enough books visible via `GET /api/books`, re-running changes nothing, and total DashScope spend is ~0.

### WS2 — High-definition covers (real cover field) Add a `cover_key` (object-store) + presigned `cover_url` to the `Book` model + response (new migration). Build `fetch_hd_covers.py`: for each catalog book, fetch the **highest-resolution cover** available (Open Library `-L`, Google Books, Wikimedia, or the EPUB's own cover upscaled), store at `covers/{book_id}`, and serve via `GET /api/books/{id}/cover`. Covers must be crisp on Retina (the shelf renders covers ~150px but at 2–3×). Provide a tasteful generated fallback (typographic cover from title/author over a token-driven gradient) when no cover is found. Acceptance: ≥95% of the 100 books have a real HD cover; the rest have a beautiful generated one; none are broken/blank.

### WS3 — 'Upload your own EPUB' (first-class) Build `UploadBook.tsx`: drag-and-drop (and the existing Electron `Cmd+O` file picker path) for EPUB/PDF, with progress, validation (type/size/page-cap with friendly errors), live ingest-status polling (`status`/`progress`/`stage` already exist on `BookResponse`), and an optimistic shelf placeholder that resolves to the real book. Wire `uploadBook` through `library.ts`. Handle the unhappy paths: oversized file, >300 pages, corrupt EPUB, duplicate. Acceptance: a user can drag an arbitrary EPUB onto the library and watch it appear, ingest, and become openable — end to end, no console errors.

### WS4 — Library UI worthy of the catalogue Rebuild `LibraryPage`/`BookShelf`/`BookCard` for 100+ books: genre/era **shelves**, **search** (title/author), filter chips, sort, 'continue reading,' and a rich `BookCard` (HD cover, author, progress, LIVE/NEW badges, genre tag) — all consuming Agent 4's shelf physics + tilt and Agent 8's tokens. Replace the mock `data/books.ts` reliance with the real backend library (keep a typed local mock only as an offline fallback). Performance with 100+ covers must stay 60fps (virtualize rows / lazy-load cover images). Acceptance: the library loads 100+ real books with HD covers, scrolls at 60fps, and search/filter work. ---
